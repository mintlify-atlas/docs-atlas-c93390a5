H2O Sonar can evaluate LLMs and RAG systems hosted by a wide range of services. Use `ConnectionConfig` to configure each host before running an evaluation.

## Token security

The `token` and `password` fields in `ConnectionConfig` are automatically encrypted at rest using AES encryption. Set the `H2O_SONAR_ENCRYPTION_KEY` environment variable to use a custom encryption key:

```bash
export H2O_SONAR_ENCRYPTION_KEY="your-32-character-secret-key-here"
```

<Warning>
  Never hard-code API keys or tokens in source code. Load them from environment variables or a secrets manager at runtime.
</Warning>

```python
import os
from h2o_sonar import config as h2o_sonar_config

token = os.environ["MY_API_TOKEN"]

model_host = h2o_sonar_config.ConnectionConfig(
    connection_type=h2o_sonar_config.ConnectionConfigType.OPENAI_CHAT.name,
    name="OpenAI",
    description="OpenAI Chat API.",
    server_url="https://api.openai.com/v1/",
    token=token,
    token_use_type=h2o_sonar_config.TokenUseType.API_KEY.name,
)
```

## `TokenUseType` values

| Value | Description |
|-------|-------------|
| `API_KEY` | Static API key issued by the service. |
| `ACCESS_TOKEN` | Short-lived personal access token. |
| `REFRESH_TOKEN` | Long-lived refresh token (used with Keycloak or H2O token provider). |
| `SECRET` | Client secret for client credentials flow. |

## Persisting connection configurations

`ConnectionConfig` can be serialized to and from a JSON dictionary. Sensitive fields (`token`, `password`) are encrypted automatically:

```python
import json
from h2o_sonar import config as h2o_sonar_config

# serialize (tokens are AES-encrypted by default)
config_dict = model_host.to_dict(encrypt=True)
with open("connection.json", "w") as f:
    json.dump(config_dict, f)

# deserialize (tokens are decrypted using H2O_SONAR_ENCRYPTION_KEY)
with open("connection.json") as f:
    config_dict = json.load(f)
model_host = h2o_sonar_config.ConnectionConfig.from_dict(config_dict)
```
