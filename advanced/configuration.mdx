## Config file format

H2O Sonar accepts configuration files in **JSON** or **TOML** format. The config file path is passed via the `--config-path` CLI flag or the `h2o_sonar_config_path` argument in Python. Only the fields you want to override are required — all other settings retain their defaults.

A minimal JSON config file with a single connection:

```json h2o-sonar-config.json
{
  "h2o_host": "localhost",
  "h2o_port": 12349,
  "h2o_auto_start": true,
  "connections": [
    {
      "key": "4cff6fc9-f49a-4dda-aeb5-2c42e9f12807",
      "connection_type": "H2O_GPT_E",
      "name": "H2O GPT Enterprise",
      "description": "H2O GPT Enterprise service.",
      "server_url": "https://h2ogpte.h2o.ai",
      "token": "YOUR_API_TOKEN_HERE",
      "token_use_type": "API_KEY"
    }
  ]
}
```

## ConnectionConfig fields

Each entry in the `connections` array is a `ConnectionConfig` object with the following fields:

<ParamField path="key" type="str">
  Unique identifier for the connection (UUID4). Auto-generated if not provided. Used to reference the connection in other settings such as `--upload-to`.
</ParamField>

<ParamField path="connection_type" type="str" required>
  The type of connection. See [connection types](#connection-types) below for valid values.
</ParamField>

<ParamField path="name" type="str" required>
  A human-readable name for the connection.
</ParamField>

<ParamField path="description" type="str" required>
  A description of what this connection is used for.
</ParamField>

<ParamField path="server_url" type="str">
  Base URL of the remote service, for example `https://h2ogpte.h2o.ai`.
</ParamField>

<ParamField path="server_id" type="str">
  Server identifier used when the server does not have a fixed URL (for example, a Driverless AI engine name in H2O AIEM).
</ParamField>

<ParamField path="token" type="str">
  API key, access token, refresh token, or client secret. Stored encrypted when an encryption key is configured. See `token_use_type` for disambiguation.
</ParamField>

<ParamField path="token_use_type" type="str">
  How the `token` field is used:
  - `API_KEY` — token is an API key passed in request headers.
  - `ACCESS_TOKEN` — token is a personal access token.
  - `REFRESH_TOKEN` — token is a client refresh token (used with Keycloak-based auth).
  - `SECRET` — token is a client secret.
</ParamField>

<ParamField path="auth_server_url" type="str">
  Authentication server URL (used for Keycloak-based authentication flows).
</ParamField>

<ParamField path="environment_url" type="str">
  H2O.ai environment URL used in the H2O token provider authentication method, for example `https://cloud.h2o.ai/`.
</ParamField>

<ParamField path="realm_name" type="str">
  Keycloak realm name (authentication domain) or workspace name for H2O AIEM connections.
</ParamField>

<ParamField path="client_id" type="str">
  OAuth2 client ID.
</ParamField>

<ParamField path="username" type="str">
  Username for username/password authentication flows.
</ParamField>

<ParamField path="password" type="str">
  Password. Stored encrypted when an encryption key is configured.
</ParamField>

## Connection types

The `connection_type` field accepts one of the following string values from `ConnectionConfigType`:

| Value | Description |
|-------|-------------|
| `H2O_GPT_E` | H2O GPT Enterprise (h2oGPTe) |
| `H2O_GPT` | H2O GPT (open-source) |
| `H2O_LLM_OPS` | H2O LLMOps |
| `OPENAI_CHAT` | OpenAI Chat API |
| `OPENAI_RAG` | OpenAI Assistants with File Search (RAG) |
| `AZURE_OPENAI_CHAT` | Microsoft Azure OpenAI Chat API |
| `ANTHROPIC_CHAT` | Anthropic Claude Chat API |
| `AMAZON_BEDROCK` | Amazon Bedrock |
| `OLLAMA` | ollama |
| `HF_SPACES` | HuggingFace Spaces API |
| `DRIVERLESS_AI` | Standalone H2O Driverless AI |
| `DRIVERLESS_AI_STEAM` | Driverless AI in H2O Enterprise Steam |
| `DRIVERLESS_AI_AIEM` | Driverless AI in H2O AI Engine Manager |
| `H2O_3` | H2O-3 |

## Adding connections

<CodeGroup>

```bash CLI
h2o-sonar add config \
  --config-path h2o-sonar-config.json \
  --config-type CONNECTION \
  --config-value '{
    "connection_type": "H2O_GPT_E",
    "name": "H2O GPT Enterprise",
    "description": "Production h2oGPTe instance.",
    "server_url": "https://h2ogpte.h2o.ai",
    "token": "YOUR_API_TOKEN_HERE",
    "token_use_type": "API_KEY"
  }'
```

```python Python
from h2o_sonar import interpret
from h2o_sonar import config as h2o_sonar_config

connection = h2o_sonar_config.ConnectionConfig(
    connection_type=h2o_sonar_config.ConnectionConfigType.H2O_GPT_E.name,
    name="H2O GPT Enterprise",
    description="Production h2oGPTe instance.",
    server_url="https://h2ogpte.h2o.ai",
    token="YOUR_API_TOKEN_HERE",
    token_use_type=h2o_sonar_config.TokenUseType.API_KEY.name,
)

interpret.add_config_item(
    config_type=h2o_sonar_config.ConfigItemType.CONNECTION,
    config_value=connection.to_dict(encrypt=False),
)
```

</CodeGroup>

## Encryption

Sensitive config fields (`token` and `password`) are automatically encrypted when an encryption key is present. Encrypted values are stored as:

```json
{
  "token": {
    "encrypted": "gAAA3LcKQ7x_X...gnKsBqVdNydYTlk8nyQ=="
  }
}
```

Set the encryption key via the environment variable (recommended) or the `--encryption-key` CLI flag:

```bash
export H2O_SONAR_ENCRYPTION_KEY="your-secret-key"
```

<Warning>
  Do not commit plaintext tokens or passwords to version control. Use environment variables or a secrets manager to supply the encryption key at runtime.
</Warning>

## H2O-3 settings

The following top-level keys configure the H2O-3 backend used by predictive explainers:

| Key | Default | Description |
|-----|---------|-------------|
| `h2o_host` | `localhost` | H2O-3 server hostname. |
| `h2o_port` | `12349` | H2O-3 server port. |
| `h2o_auto_start` | `true` | Start H2O-3 automatically if not running. |
| `h2o_min_mem_size` | `2G` | Minimum JVM heap size for H2O-3. |
| `h2o_max_mem_size` | `4G` | Maximum JVM heap size for H2O-3. |

## GPU acceleration

Set the `H2O_SONAR_CFG_DEVICE` environment variable to enable GPU acceleration for evaluators such as BERTScore, GPTScore, and Perplexity:

```bash
export H2O_SONAR_CFG_DEVICE="gpu"
```

The default value is `auto`, which detects available hardware automatically. Valid values are `cpu`, `gpu`, or `auto`.

<Tip>
  GPU support requires a CUDA-compatible GPU with NVIDIA drivers and the `[evaluators]` extras installed: `pip install h2o_sonar-<version>.whl[evaluators]`.
</Tip>
